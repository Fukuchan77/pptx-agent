# サンプル入力ファイル

このディレクトリには、AI PowerPoint Presentation Generatorのさまざまなユースケースを示すサンプル入力ファイルが含まれています。これらのサンプルは、異なるコンテンツタイプ、言語、および視覚化ニーズにわたるシステムの機能を紹介しています。

## 利用可能なサンプル

### 01-business-quarterly-review.txt

**タイプ**: ビジネスプレゼンテーション（英語）
**内容**: 財務パフォーマンス、顧客指標、販売実績、市場ポジション、および業務の卓越性をカバーする包括的な四半期ビジネスレビュー
**対象者**: 経営幹部、取締役会メンバー、投資家
**推奨テンプレート**: `basic-template.pptx` または `japanese-template.pptx`
**予想スライド数**: 15-20枚
**特徴**: エグゼクティブサマリー、財務指標、戦略的イニシアチブ、KPI

**使用方法**:

```bash
python -m pptx_agent.main \
  --input examples/01-business-quarterly-review.txt \
  --template templates/basic-template.pptx \
  --output output/quarterly-review.pptx
```

**期待される内容**:

- プレゼンテーション概要を含むタイトルスライド
- 主要トピックのセクションヘッダー
- 主要指標の箇条書きを含むコンテンツスライド
- 戦略的イニシアチブと優先事項
- 次のステップを含む結論

## 一般的な使用手順

### 基本的なコマンド構造

```bash
python -m pptx_agent.main \
  --input <入力ファイルパス> \
  --template <テンプレートファイルパス> \
  --output <出力ファイルパス> \
  [--language ja|en]
```

### パラメータ

- `--input`: 入力テキストファイルへのパス（必須）
- `--template`: PowerPointテンプレートファイルへのパス（必須）
- `--output`: 生成されるプレゼンテーションファイルのパス（必須）
- `--language`: 出力言語 - 日本語は `ja`、英語は `en`（任意、指定しない場合は自動検出）

### 言語検出

システムは入力言語を自動的に検出しますが、出力言語を明示的に指定することもできます:

**英語出力**:

```bash
python -m pptx_agent.main \
  --input examples/01-business-quarterly-review.txt \
  --template templates/basic-template.pptx \
  --output output/presentation-en.pptx \
  --language en
```

**日本語出力**:

```bash
python -m pptx_agent.main \
  --input examples/03-python-programming-basics-ja.txt \
  --template templates/japanese-template.pptx \
  --output output/presentation-ja.pptx \
  --language ja
```

### テンプレート選択ガイド

| コンテンツタイプ                | 推奨テンプレート           | 理由                                   |
| ------------------------------- | -------------------------- | -------------------------------------- |
| ビジネスプレゼンテーション（英語） | `basic-template.pptx`      | 汎用レイアウト、英語フォント           |
| 技術プレゼンテーション（英語）     | `basic-template.pptx`      | 技術コンテンツ用の明確なレイアウト     |
| 教育コンテンツ（日本語）           | `japanese-template.pptx`   | 日本語フォント、最適化された容量       |
| データ視覚化                       | `data-template.pptx`\*     | グラフと表のレイアウト                 |
| プロセスフロー                     | `smartart-template.pptx`\* | SmartArt図                             |

\*アスタリスク付きのテンプレートは手動作成が必要です（仕様は `templates/` ディレクトリで利用可能）

## 最良の結果を得るためのヒント

### コンテンツ構造

1. **明確な見出し**: コンテンツを論理的なスライドに整理するために明確なセクション見出しを使用します
2. **簡潔な段落**: 段落を1つの主要なアイデアに焦点を当てます
3. **重要なポイント**: 重要な指標、日付、および決定を強調します
4. **論理的なフロー**: 導入から結論までの物語の流れでコンテンツを構造化します

### スライド数のガイドライン

- **短いプレゼンテーション**: 1,000-2,000文字 → 5-10枚
- **中程度のプレゼンテーション**: 2,000-5,000文字 → 10-15枚
- **長いプレゼンテーション**: 5,000-10,000文字 → 15-25枚
- **最大推奨**: 30,000文字 → 約30枚（20枚を超えると品質が低下する可能性があります）

### 言語固有のヒント

**日本語コンテンツの場合**:

- 適切なフォントサポートのために `japanese-template.pptx` を使用します
- 正確なテキスト容量計算のために `--language ja` を指定します
- 日本語テキストは英語の約1.8倍のスペースが必要です（自動的に処理されます）
- 漢字、ひらがな、カタカナの混在が正しく処理されます

**英語コンテンツの場合**:

- ほとんどのユースケースで `basic-template.pptx` がうまく機能します
- 技術用語と頭字語が適切に処理されます
- 箇条書きが自動的にフォーマットされます

### データと視覚化

**数値データ**:

- 明確なラベルを持つ指標を含めます: "収益: $10.5M" または "成長: 25%"
- 比較とトレンドを含むコンテキストでデータを提示します
- AIはスライド内でデータを適切に構造化します

**表**（data-template.pptx が利用可能な場合）:

- ヘッダーと行を含む表形式のデータを明確に説明します
- 表を最大20行×10列に保ちます
- AIはテキストの説明をフォーマットされた表に変換します

**グラフ**（data-template.pptx が利用可能な場合）:

- 数値関係とトレンドを説明します
- 必要に応じてグラフタイプを指定します（棒、線、円など）
- AIはグラフに適したデータを識別し、視覚化を作成します

**プロセスフロー**（smartart-template.pptx が利用可能な場合）:

- 順次ステップを明確に説明します
- 番号付きリストまたはフェーズを使用します
- AIはSmartArt図に変換します

## 独自のサンプルの作成

### コンテンツガイドライン

1. **明確な目的から始める**: プレゼンテーションが伝えるべきことを定義します
2. **主要トピックの概要**: 3-7の主要セクションをリストします
3. **各セクションを展開**: セクションごとに2-4段落を書きます
4. **サポート詳細を追加**: 指標、例、およびコンテキストを含めます
5. **要点で締めくくる**: 重要なポイントと次のステップを要約します

### コンテンツ長の推奨事項

- **最小**: 1,000文字（仕様FR-005に従い、最低10行）
- **最適**: 10-20枚のスライドプレゼンテーションの場合、3,000-8,000文字
- **最大**: 30,000文字（仕様FR-005に従う厳密な制限）

### 執筆スタイルのヒント

**ビジネスプレゼンテーションの場合**:

- プロフェッショナルで簡潔な言語を使用します
- 具体的な指標とデータポイントを含めます
- エグゼクティブサマリー、分析、推奨事項で構造化します
- 明確なアクションアイテムまたは次のステップで終わります

**技術プレゼンテーションの場合**:

- 正確な技術用語を使用します
- 複雑な概念を論理的な進行で説明します
- アーキテクチャ、実装、および運用の詳細を含めます
- 混合オーディエンスのために深さとアクセシビリティのバランスを取ります

**教育コンテンツの場合**:

- 基礎から始めて、徐々に複雑さを構築します
- 概念を説明するために例と類推を使用します
- 実践シナリオまたはユースケースを含めます
- 明確な学習目標で構造化します

**データ駆動型プレゼンテーションの場合**:

- 洞察でリードし、データでサポートします
- コンテキスト（トレンド、比較）を含む具体的な数値を使用します
- データの進行でストーリーを語ります
- 指標からの重要な要点を強調します

## サンプルワークフロー

### クイックテスト実行

短いサンプルでシステムをテストします:

```bash
# プレゼンテーションを生成
python -m pptx_agent.main \
  --input examples/01-business-quarterly-review.txt \
  --template templates/basic-template.pptx \
  --output test-output.pptx

# PowerPointまたはLibreOffice Impressで開く
open test-output.pptx  # macOS
start test-output.pptx  # Windows
xdg-open test-output.pptx  # Linux
```

### 複数のサンプルのバッチ処理

すべてのサンプルを生成します:

```bash
# 出力ディレクトリを作成
mkdir -p output

# 各サンプルを生成
for example in examples/*.txt; do
    filename=$(basename "$example" .txt)
    python -m pptx_agent.main \
        --input "$example" \
        --template templates/basic-template.pptx \
        --output "output/${filename}.pptx"
    echo "生成完了: output/${filename}.pptx"
done
```

### 日本語サンプルワークフロー

## トラブルシューティング

### 問題: 生成されたプレゼンテーションのスライド数が多すぎる

**解決策**: 入力テキストの長さを減らすか、複数のプレゼンテーションに分割します。最適範囲は10-20枚です（仕様SC-013に従い、20枚を超えると品質が低下する可能性があります）。

### 問題: スライドのテキストがはみ出す

**解決策**:

- システムは段階的な解決を通じてオーバーフローを自動的に処理します（仕様FR-032に従い、フォント縮小 → レイアウト変更 → スライド分割 → 要約）
- 言語に適したテンプレートを使用します（日本語テキストには日本語テンプレート）
- `--language` フラグで正しい言語を指定します

### 問題: 日本語文字が正しく表示されない

**解決策**:

- `japanese-template.pptx` テンプレートを使用します
- `--language ja` パラメータを指定します
- 入力ファイルがUTF-8でエンコードされていることを確認します

### 問題: プレゼンテーションが期待される構造と一致しない

**解決策**:

- 入力テキストに明確なセクション見出しがあることを確認します
- コンテンツが論理的なフローで適切に構造化されていることを確認します
- コンテンツ構造の理解のために生成されたアウトラインを確認します

## パフォーマンスの期待値

成功基準（spec.md）に基づく:

- **テンプレート解析**: 典型的なテンプレートで5秒未満（SC-002）
- **アウトライン生成**: 120秒のタイムアウト（FR-047）
- **スライドコンテンツ生成**: スライドあたり60秒のタイムアウト（FR-048）
- **PowerPointレンダリング**: コンテンツ生成後10秒未満（SC-003）
- **合計生成**: LLM時間を除く10枚のスライドプレゼンテーションで60秒未満を目標（SC-001）

**注意**: 実際の生成時間は以下に依存します:

- 入力テキストの長さと複雑さ
- 生成されるスライド数
- LLMプロバイダーの応答時間
- テンプレートの複雑さ
